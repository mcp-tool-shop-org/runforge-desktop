# Beta Readiness Checklist — RunForge Desktop v0.9.0-rc.1

This checklist certifies that RunForge Desktop is ready for public beta release.

## Release Criteria

### 1. Code Quality
- [x] All code compiles without errors
- [x] No critical or high-severity warnings
- [x] Code follows established patterns (MVVM, DI)
- [x] No hard-coded secrets or credentials

### 2. Testing
- [x] Unit tests pass (339+ tests in RunForgeDesktop.Core.Tests)
- [x] Button coverage inventory complete (44 buttons documented)
- [x] Soak test harness available (`scripts/soak-test.ps1`)
- [ ] 2-hour soak test completed without issues (manual)
- [ ] VM install/upgrade/uninstall tested (manual)

### 3. Documentation
- [x] README.md exists and is current
- [x] CHANGELOG.md follows Keep a Changelog format
- [x] SECURITY.md documents vulnerability reporting
- [x] LICENSE file present (MIT)
- [x] Help tab has FAQ and troubleshooting
- [x] VM install runbook documented

### 4. User Experience
- [x] Light theme renders correctly
- [x] Dark theme renders correctly
- [x] No hard-coded colors (AppThemeBinding used)
- [x] WCAG AA contrast compliance documented
- [x] Crash recovery dialog implemented
- [x] Version displayed in title bar

### 5. Release Infrastructure
- [x] GitHub repo has issue templates
- [x] GitHub repo has PR template
- [x] Release bundle script exists
- [x] SHA256 checksums generated by script
- [x] MSIX packaging configured

### 6. Known Limitations (Acceptable for Beta)
- [ ] MSIX not code-signed (requires certificate purchase)
- [ ] CI/CD pipeline not configured
- [ ] Telemetry/analytics not implemented
- [ ] Auto-update mechanism not implemented
